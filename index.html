The Library That Makes This Possible

NotebookLM does not have an official public API. This system uses an open-source Python library called notebooklm-py built by Tang Lin that acts as a bridge by automating the NotebookLM web interface using a real browser session in the background.

The library is community-built and actively maintained. Because it is unofficial, Google could update NotebookLM in the future and require a library update. For now it works reliably and is the only way to connect NotebookLM to Claude Code programmatically.

Where to Paste

Every code block in this guide is labeled with one of three locations. Here is what each one means.

Cursor Terminal

The terminal panel at the bottom of Cursor. Open it by pressing Ctrl + ` (Control and the backtick key, top left of your keyboard). This is where most install commands go.

Mac Terminal

The standalone Terminal app on your Mac. Open it by pressing Cmd + Space, typing Terminal, and hitting Enter. Only two steps in this guide require the Mac Terminal — both involve Google authentication which cannot run inside Cursor.

Claude Code Sidebar

The chat panel on the right side of Cursor where you talk to Claude Code. This is where you paste prompts that ask Claude to build skills and run commands on your behalf.

Into a File

A blank file that Cursor opens for you. Paste the content in and save with Cmd + S.

Part 01

Install yt-search

Step 01
Open the Cursor terminal

Press Ctrl + ` (Control and the backtick key, top left of your keyboard). The terminal panel opens at the bottom of Cursor.

Step 02
Install yt-dlp

yt-dlp is the tool that searches YouTube behind the scenes. The skill calls it automatically.

This tells your Mac where to find yt-dlp when it runs.

echo 'export PATH="$HOME/Library/Python/3.9/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

Press 1 when asked for approval.

Step 06
Open the search script
Cursor Terminal 
code ~/.claude/skills/yt-search/scripts/search.py

A blank file opens. Paste the full script below into it and save with Cmd + S.

Into the File 
#!/usr/bin/env python3
import io, json, shutil, subprocess, sys
from datetime import datetime, timedelta

sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding="utf-8", errors="replace")
sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding="utf-8", errors="replace")

def parse_args(argv):
    args = argv[1:]; count = 20; months = 6; query_parts = []; i = 0
    while i < len(args):
        if args[i] == "--count" and i + 1 < len(args):
            count = int(args[i + 1]); i += 2
        elif args[i] == "--months" and i + 1 < len(args):
            months = int(args[i + 1]); i += 2
        elif args[i] == "--no-date-filter":
            months = 0; i += 1
        else:
            query_parts.append(args[i]); i += 1
    query = " ".join(query_parts)
    if not query:
        print("Usage: search.py <query> [--count N] [--months N] [--no-date-filter]", file=sys.stderr); sys.exit(1)
    return query, count, months

def fmt_subs(n):
    if n is None: return "N/A"
    if n >= 1_000_000: return f"{n/1_000_000:.1f}M"
    if n >= 1_000: return f"{n/1_000:.1f}K"
    return str(n)

def fmt_views(n): return "N/A" if n is None else f"{n:,}"

def fmt_dur(info):
    if info.get("duration_string"): return info["duration_string"]
    dur = info.get("duration")
    if dur is None: return "N/A"
    dur = int(dur); h, r = divmod(dur, 3600); m, s = divmod(r, 60)
    return f"{h}:{m:02d}:{s:02d}" if h else f"{m}:{s:02d}"

def fmt_date(raw):
    if not raw or len(raw) != 8: return "N/A"
    try: return datetime.strptime(raw, "%Y%m%d").strftime("%b %d, %Y")
    except: return f"{raw[:4]}-{raw[4:6]}-{raw[6:8]}"

def main():
    query, count, months = parse_args(sys.argv)
    if not shutil.which("yt-dlp"):
        print("Error: yt-dlp not found. Install: pip install yt-dlp", file=sys.stderr); sys.exit(1)
    fetch = count * 2 if months > 0 else count
    cmd = ["yt-dlp", f"ytsearch{fetch}:{query}", "--dump-json", "--no-download", "--no-warnings", "--quiet"]
    try:
        r = subprocess.run(cmd, capture_output=True, text=True, timeout=120)
    except subprocess.TimeoutExpired:
        print("Error: Search timed out.", file=sys.stderr); sys.exit(1)
    if r.returncode != 0 and not r.stdout.strip():
        print(f"Error: {r.stderr.strip()}", file=sys.stderr); sys.exit(1)
    videos = []
    for line in r.stdout.strip().splitlines():
        try: videos.append(json.loads(line))
        except: continue
    if not videos:
        print("No results found.", file=sys.stderr); sys.exit(0)
    if months > 0:
        cutoff = (datetime.now() - timedelta(days=months*30)).strftime("%Y%m%d")
        filtered = [v for v in videos if (v.get("upload_date") or "00000000") >= cutoff]
        skipped = len(videos) - len(filtered); videos = filtered
        if skipped: print(f"(Filtered {skipped} older video(s))\n", file=sys.stderr)
    if not videos:
        print(f"No results in last {months} months.", file=sys.stderr); sys.exit(0)
    videos = videos[:count]; d = "\u2500"*60
    for i, v in enumerate(videos, 1):
        url = f"https://youtube.com/watch?v={v.get('id','')}"
        meta = f"{v.get('channel','?')} ({fmt_subs(v.get('channel_follower_count'))}) · {fmt_views(v.get('view_count'))} views · {fmt_dur(v)} · {fmt_date(v.get('upload_date',''))}"
        print(d); print(f" {i:>2}. {v.get('title','?')}"); print(f"     {meta}"); print(f"     {url}")
    print(d)

if __name__ == "__main__":
    main()

Part 02

Install NotebookLM

Step 08
Install Python 3.12

NotebookLM-py requires Python 3.10 or newer. Most Macs ship with 3.9 which is too old. This installs a newer version without breaking anything.

Cursor Terminal 
brew install python@3.12

Takes a few minutes. Approve any prompts with 1.

Step 09
Install the notebooklm-py library
Cursor Terminal 
pipx install "notebooklm-py[browser]" --python /opt/homebrew/bin/python3.12 --force

Then install the browser component:

Cursor Terminal 
~/.local/pipx/venvs/notebooklm-py/bin/playwright install chromium

Approve both with 1. Wait for each to finish.

Step 10
Fix the PATH

This ensures the correct version of notebooklm runs every time you open a terminal.

Cursor Terminal 
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

Remove the old version if it exists:

Cursor Terminal 
rm ~/Library/Python/3.9/bin/notebooklm 2>/dev/null

Verify the correct version is running:

Cursor Terminal 
notebooklm --version

You should see NotebookLM CLI, version 0.3.4 or higher. If you see a version number, continue.

Step 11
Authenticate with Google

This step must run in the Mac Terminal app, not the Cursor terminal. The login opens a browser window that Claude Code cannot handle.

Press Cmd + Space, type Terminal, hit Enter to open the Mac Terminal app. Then run:

Mac Terminal 
notebooklm login

A browser window opens. Log into your Google account, go to notebooklm.google.com, wait until you see the NotebookLM homepage, then come back to the Mac Terminal window and press Enter.

You will see a confirmation that your authentication was saved.

Step 12
Install the NotebookLM skill into Claude Code
Cursor Terminal 
~/.local/bin/notebooklm skill install

Verify both skills are installed:

Cursor Terminal 
ls ~/.claude/skills/

You should see yt-search and notebooklm listed. If both are there, continue.

Part 03

Build /research-topic

Step 13
Build the skill

This creates the command that chains yt-search and NotebookLM into one automated step. Claude Code builds it for you.

Open a new session in the Claude Code sidebar and paste the entire prompt below:

Claude Code Sidebar 
Use the new-capability skill to build a skill called /research-topic that does the following:

1. Takes a search query as input
2. Uses yt-search to find the top 10 YouTube videos on that topic from the last 6 months
3. Automatically creates a NEW NotebookLM notebook named after the search query
4. Loads all the YouTube URLs as sources into that notebook
5. Runs NotebookLM deep web research on the same topic automatically
6. Ranks and filters the deep research results down to the top 10 highest quality web sources
7. Adds those top 10 sources to the notebook alongside the YouTube sources
8. Confirms when complete with a breakdown of YouTube and web sources loaded

Default behavior is always a new notebook. Adding to an existing notebook only happens when I use --add-to.

Approve all permission prompts with 2 as they appear. This takes a few minutes to build.

Step 14
Test the full system

Open a new Claude Code session and run the command below. You should see it search YouTube, create a notebook, load sources, run deep research, filter to the top 10 web sources, and confirm with a full breakdown. The process takes 5 to 10 minutes.

Claude Code Sidebar 
/research-topic AI transformation consulting

Pathway 02

Research to Build

This is where most AI workflows stop. Yours is where they start building.

The notebook you built in Part 3 contains curated, ranked, real-world knowledge on your topic. Instead of letting it sit there, you are going to feed it directly into Claude and build a skill that operates from that knowledge permanently. Every time you use the skill, Claude already knows the domain. You never have to re-explain the context.

Step 1 — Get a structured synthesis from your notebook

Before building the skill, ask NotebookLM to synthesize what it learned. Open a new Claude Code session and paste this, replacing the notebook name with yours:

Claude Code Sidebar 
Ask the "[your notebook name]" notebook: what are the core principles,
frameworks, and best practices that come up most consistently across
all sources? Give me a structured breakdown I can use to build a
Claude skill around this domain.

Copy the full response. This becomes the knowledge base for your skill.

Step 2 — Build the skill

Open a new Claude Code session. Paste the prompt below, then paste your notebook synthesis directly underneath it before hitting Enter:

Claude Code Sidebar 
I want to create a Claude skill for [your domain]. Here is the
research and knowledge base I want this skill to work from:

[paste your NotebookLM synthesis here]

Build a skill that:
- Activates automatically when I ask for help with [your domain]
- Uses this knowledge base to inform every response
- Follows the frameworks and best practices from the research
- Produces outputs in [describe your preferred format]

Ask me any clarifying questions before building.

Approve all permission prompts with 2 as they appear.

Step 3 — Answer Claude's three questions

Claude Code will ask three things before it builds. Answer each one directly in the sidebar.

1
What output types do you need?
Example: cold emails, LinkedIn messages, full outreach sequences, audit reports, proposals. Be specific about what you want the skill to produce.
2
What inputs should Claude expect?
Example: prospect name and company, client context and industry, product description. Tell it what information you will give it each time you use the skill.
3
How opinionated should the skill be?
Very rigid (always follow the exact framework), flexible (suggest the framework but adapt to context), or loose (apply the knowledge without enforcing structure). Flexible works best for most use cases.
Step 4 — Test your new skill

Once Claude Code finishes building, open a new session, select your new skill, and give it a real scenario from your work:

Claude Code Sidebar 
Use the [skill name] skill. Here is the context:
[describe a real situation from your work]

You should get back a response that reflects the actual frameworks from your research — not generic AI output. The difference is immediate and noticeable.

How to make your skill smarter over time

Your skill compounds. Every time you run /research-topic on a related topic and synthesize the results, you can update the skill with new knowledge. Run this in a new Claude Code session:

Claude Code Sidebar 
/research-topic [related topic] --add-to "[existing notebook name]"

Then ask the updated notebook for a new synthesis and update your skill file. The agent gets sharper with every notebook you add. This is what it means when intelligence compounds.

Troubleshooting

If Something Goes Wrong

Do not guess. Ask Claude.

Every machine is different. Instead of a list of pre-written error fixes that may or may not match what you see, use this prompt. It works for any error you hit during this install — not just the ones someone predicted.

Copy the exact error message that appeared on your screen, paste it into the prompt below, and send it to Claude in a new Claude Code session.

Claude Code Sidebar 
I am installing a YouTube search and NotebookLM research system
in Claude Code on a Mac. I got this error:

[paste the exact error message here]

Tell me exactly what caused this and give me the step by step fix.

Claude will read your exact error, diagnose the specific cause, and give you the fix for your machine. This is more reliable than any pre-written list because it handles whatever you actually encounter rather than whatever someone else predicted.

One thing worth knowing before you hit an error: the two most common issues during this install are PATH problems (your Mac cannot find a tool that is already installed) and Python version conflicts (a command ran on the wrong Python version). Both are fixable in under two minutes once you know which one it is. Claude will identify it immediately from your error message.
