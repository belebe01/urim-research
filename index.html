What You Are Actually Installing

Before you touch a single command, here is the plain English picture. You are connecting three tools that already exist and are all free. Once connected, one command runs all three automatically.

🎬
yt-dlp
Searches YouTube and returns the top videos on any topic. Filters by date so you only get recent, relevant content.
📓
NotebookLM
Google's free research tool. Stores your sources, runs deep web research across hundreds of sites, and generates content from what it finds.
🤖
Claude Code
The AI brain running inside Cursor. It orchestrates everything — calling yt-dlp, talking to NotebookLM, and building skills from your research.
→

When you type /research-topic [anything] in Claude Code, it searches YouTube, creates a NotebookLM notebook, loads the best sources, runs deep web research, filters to the top 10 results, and confirms when everything is ready. You do nothing except type the topic.

One Thing Worth Knowing Before You Start

NotebookLM does not have an official public API. This system uses an open-source Python library called notebooklm-py built by Tang Lin that connects to NotebookLM by automating the web interface using a real browser session running in the background. Think of it as a robot that uses NotebookLM the same way you would — it just does it automatically.

The library is community-built and actively maintained. It works reliably and is the only way to connect NotebookLM to Claude Code programmatically.

github.com/teng-lin/notebooklm-py →

Install yt-search

Open the Cursor terminal

Step 02
Install yt-dlp

Step 06
Open the search script
Cursor Terminal 
code ~/.claude/skills/yt-search/scripts/search.py

A blank file opens. Paste the full script below and save with Cmd + S.

Into the File 
#!/usr/bin/env python3
import io, json, shutil, subprocess, sys
from datetime import datetime, timedelta

sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding="utf-8", errors="replace")
sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding="utf-8", errors="replace")

def parse_args(argv):
    args = argv[1:]; count = 20; months = 6; query_parts = []; i = 0
    while i < len(args):
        if args[i] == "--count" and i + 1 < len(args):
            count = int(args[i + 1]); i += 2
        elif args[i] == "--months" and i + 1 < len(args):
            months = int(args[i + 1]); i += 2
        elif args[i] == "--no-date-filter":
            months = 0; i += 1
        else:
            query_parts.append(args[i]); i += 1
    query = " ".join(query_parts)
    if not query:
        print("Usage: search.py <query> [--count N] [--months N] [--no-date-filter]", file=sys.stderr); sys.exit(1)
    return query, count, months

def fmt_subs(n):
    if n is None: return "N/A"
    if n >= 1_000_000: return f"{n/1_000_000:.1f}M"
    if n >= 1_000: return f"{n/1_000:.1f}K"
    return str(n)

def fmt_views(n): return "N/A" if n is None else f"{n:,}"

def fmt_dur(info):
    if info.get("duration_string"): return info["duration_string"]
    dur = info.get("duration")
    if dur is None: return "N/A"
    dur = int(dur); h, r = divmod(dur, 3600); m, s = divmod(r, 60)
    return f"{h}:{m:02d}:{s:02d}" if h else f"{m}:{s:02d}"

def fmt_date(raw):
    if not raw or len(raw) != 8: return "N/A"
    try: return datetime.strptime(raw, "%Y%m%d").strftime("%b %d, %Y")
    except: return f"{raw[:4]}-{raw[4:6]}-{raw[6:8]}"

def main():
    query, count, months = parse_args(sys.argv)
    if not shutil.which("yt-dlp"):
        print("Error: yt-dlp not found. Install: pip install yt-dlp", file=sys.stderr); sys.exit(1)
    fetch = count * 2 if months > 0 else count
    cmd = ["yt-dlp", f"ytsearch{fetch}:{query}", "--dump-json", "--no-download", "--no-warnings", "--quiet"]
    try:
        r = subprocess.run(cmd, capture_output=True, text=True, timeout=120)
    except subprocess.TimeoutExpired:
        print("Error: Search timed out.", file=sys.stderr); sys.exit(1)
    if r.returncode != 0 and not r.stdout.strip():
        print(f"Error: {r.stderr.strip()}", file=sys.stderr); sys.exit(1)
    videos = []
    for line in r.stdout.strip().splitlines():
        try: videos.append(json.loads(line))
        except: continue
    if not videos:
        print("No results found.", file=sys.stderr); sys.exit(0)
    if months > 0:
        cutoff = (datetime.now() - timedelta(days=months*30)).strftime("%Y%m%d")
        filtered = [v for v in videos if (v.get("upload_date") or "00000000") >= cutoff]
        skipped = len(videos) - len(filtered); videos = filtered
        if skipped: print(f"(Filtered {skipped} older video(s))\n", file=sys.stderr)
    if not videos:
        print(f"No results in last {months} months.", file=sys.stderr); sys.exit(0)
    videos = videos[:count]; d = "\u2500"*60
    for i, v in enumerate(videos, 1):
        url = f"https://youtube.com/watch?v={v.get('id','')}"
        meta = f"{v.get('channel','?')} ({fmt_subs(v.get('channel_follower_count'))}) · {fmt_views(v.get('view_count'))} views · {fmt_dur(v)} · {fmt_date(v.get('upload_date',''))}"
        print(d); print(f" {i:>2}. {v.get('title','?')}"); print(f"     {meta}"); print(f"     {url}")
    print(d)

if __name__ == "__main__":
    main()

Part 02
About 15 minutes

Install NotebookLM

This part has the most steps but every single one is a copy-paste command. Take it one step at a time. You will be through it in about 15 minutes.

Step 08
Install Python 3.12

NotebookLM-py requires Python 3.10 or newer. Most Macs ship with Python 3.9 which is too old. This installs a newer version without breaking or removing anything on your Mac.

Cursor Terminal 
brew install python@3.12

Takes a few minutes. Approve any prompts with 1.

Step 09
Install the notebooklm-py library
Cursor Terminal 
pipx install "notebooklm-py[browser]" --python /opt/homebrew/bin/python3.12 --force

Then install the browser component:

Cursor Terminal 
~/.local/pipx/venvs/notebooklm-py/bin/playwright install chromium

Approve both with 1. Wait for each to finish before running the next.

Step 10
Fix the PATH

This ensures your Mac always finds the right version of notebooklm when it runs.

Cursor Terminal 
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc && source ~/.zshrc

Remove the old version if it exists:

Cursor Terminal 
rm ~/Library/Python/3.9/bin/notebooklm 2>/dev/null

Verify the correct version is running:

Cursor Terminal 
notebooklm --version
You will know it worked when you see this

The terminal shows NotebookLM CLI, version 0.3.4 or higher. Any version number means the install succeeded. Continue to Step 11.

Step 11
Authenticate with Google

This is the one step that requires the Mac Terminal app instead of the Cursor terminal. The login opens a real browser window that Claude Code cannot handle. You only have to do this once.

Press Cmd + Space, type Terminal, hit Enter to open the Mac Terminal app. Then run this:

Mac Terminal 
notebooklm login

A browser window opens automatically. Log into your Google account, navigate to notebooklm.google.com, wait until the NotebookLM homepage fully loads, then come back to the Mac Terminal window and press Enter.

You will know it worked when you see this

The Mac Terminal shows a message saying authentication was saved to a file path on your computer. Once you see that confirmation, close the Mac Terminal and go back to Cursor.

Step 12
Install the NotebookLM skill into Claude Code
Cursor Terminal 
~/.local/bin/notebooklm skill install

Verify both skills are installed:

Cursor Terminal 
ls ~/.claude/skills/
You will know it worked when you see this

The terminal lists both yt-search and notebooklm in your skills folder. If you see both names, Part 2 is complete.

Part 03
About 5 minutes to build · 5-10 minutes to test

Build /research-topic

Step 13
Build the skill

You are not writing any code here. You are asking Claude Code to build the skill for you by describing what you want it to do. Claude does all the work.

Open a new Claude Code session in the Cursor sidebar and paste this entire prompt:

Claude Code Sidebar 
Use the new-capability skill to build a skill called /research-topic that does the following:

1. Takes a search query as input
2. Uses yt-search to find the top 10 YouTube videos on that topic from the last 6 months
3. Automatically creates a NEW NotebookLM notebook named after the search query
4. Loads all the YouTube URLs as sources into that notebook
5. Runs NotebookLM deep web research on the same topic automatically
6. Ranks and filters the deep research results down to the top 10 highest quality web sources
7. Adds those top 10 sources to the notebook alongside the YouTube sources
8. Confirms when complete with a breakdown of YouTube and web sources loaded

Default behavior is always a new notebook. Adding to an existing notebook only happens when I use --add-to.

Approve all permission prompts with 2 as they appear. This takes a few minutes to build.

Step 14
Test the full system

Open a new Claude Code session and run this command. Use any topic you are actually curious about.

Claude Code Sidebar 
/research-topic AI transformation consulting
You will know it worked when you see this

Claude Code walks through each step out loud: YouTube searched, notebook created, YouTube sources loaded, deep research triggered, sources ranked and filtered, final count confirmed. The whole process takes 5 to 10 minutes. When it finishes you will see a summary showing how many YouTube videos and web sources were loaded into the notebook. That means everything is working.

Pathway 02

Research to Build

This is where most AI workflows stop. Yours is where they start building.

The notebook you built in Part 3 contains curated, ranked, real-world knowledge on your topic. Instead of letting it sit there, feed it directly into Claude and build a skill that operates from that knowledge permanently. Every time you use the skill, Claude already knows the domain. You never re-explain the context.

Step 1 — Get a structured synthesis from your notebook

Before building the skill, ask NotebookLM to synthesize what it learned. Open a new Claude Code session and paste this, replacing the notebook name with yours:

Claude Code Sidebar 
Ask the "[your notebook name]" notebook: what are the core principles,
frameworks, and best practices that come up most consistently across
all sources? Give me a structured breakdown I can use to build a
Claude skill around this domain.

Copy the full response. This becomes the knowledge base for your skill.

Step 2 — Build the skill

Open a new Claude Code session. Paste the prompt below, then paste your notebook synthesis directly underneath it before hitting Enter:

Claude Code Sidebar 
I want to create a Claude skill for [your domain]. Here is the
research and knowledge base I want this skill to work from:

[paste your NotebookLM synthesis here]

Build a skill that:
- Activates automatically when I ask for help with [your domain]
- Uses this knowledge base to inform every response
- Follows the frameworks and best practices from the research
- Produces outputs in [describe your preferred format]

Ask me any clarifying questions before building.

Approve all permission prompts with 2 as they appear.

Step 3 — Answer Claude's three questions

Claude Code will ask three things before it builds. Answer each one directly in the sidebar.

1
What output types do you need?
Example: cold emails, LinkedIn messages, full outreach sequences, audit reports, proposals. Be specific.
2
What inputs should Claude expect?
Example: prospect name and company, client context and industry, product description. Tell it what you will give it each time you use the skill.
3
How opinionated should the skill be?
Very rigid (always follow the exact framework), flexible (suggest the framework but adapt to context), or loose (apply knowledge without enforcing structure). Flexible works best for most people.
Step 4 — Test your new skill

Once Claude Code finishes building, open a new session, select your new skill, and give it a real scenario from your work:

Claude Code Sidebar 
Use the [skill name] skill. Here is the context:
[describe a real situation from your work]
You will know it worked when you see this

The response reflects the actual frameworks and language from your research, not generic AI output. It uses the specific terminology and structure from the sources in your notebook. The difference is immediate and obvious.

How to make your skill smarter over time

Your skill compounds. Every time you run /research-topic on a related topic and synthesize the results, you can update the skill with new knowledge.

Claude Code Sidebar 
/research-topic [related topic] --add-to "[existing notebook name]"

Then ask the updated notebook for a new synthesis and update your skill file. The agent gets sharper with every notebook you add. This is what it means when intelligence compounds.

Troubleshooting

If Something Goes Wrong

Do not guess. Ask Claude.

Every Mac is different. Instead of a list of pre-written fixes that may not match what you see, use this prompt. It works for any error you hit during this install.

Copy the exact error message from your screen, paste it into the prompt below, and send it in a new Claude Code session.

Claude Code Sidebar 
I am installing a YouTube search and NotebookLM research system
in Claude Code on a Mac. I got this error:

[paste the exact error message here]

Tell me exactly what caused this and give me the step by step fix.

Claude will read your exact error, identify the specific cause, and give you the fix for your machine. This handles whatever you encounter rather than whatever someone else predicted.

One thing worth knowing: the two most common issues during this install are PATH problems (your Mac cannot find a tool that is already installed) and Python version conflicts (a command ran on the wrong Python version). Both are fixed in under two minutes once you know which one it is. Claude will identify it immediately from your error message.
